Cat-expiry

고양이 식료품 유통기한을 관리할 수 있는 서비스입니다. React 라이브러리를 사용하여 클라이언트 개발을 하였고, Firebase를 통해 DB 연동 및 배포를 하였습니다.

*로그인 필수: 하단 My Page 게스트 로그인

Skill
React, Recoil, React-query, SCSS, Firebase
Role
Front-end, UI/UX design

프론트엔드 개발

제품, 북마크 등 데이터 관리
  • rest 방식으로 Firebase에 요청하여 데이터 조회, 추가, 수정, 삭제 기능 구현
  • React-query의 invalidateQueries 메소드를 통해 데이터 refetching하여 사용
이메일 계정, 소셜 계정 로그인/로그아웃 기능
  • Firebase의 Authentication 설정을 통해 이메일 로그인 및 소셜 로그인 기능 구현
필터 및 정렬 기능 구현
  • filter 및 sort 함수를 사용하여 필터와 정렬 기능을 구현하여 편의성을 높임
검색 기능 구현
  • React-query의 select 옵션을 활용해 filter함수로 반환값을 변환하여 검색 기능 구현
자동완성 기능 구현
  • 제품 등록 시 제품명을 입력하면 북마크에 있는 데이터를 자동완성 시켜주는 기능 구현
i18n 라이브러리를 이용한 다국어 모드 (한국어, 영어)
  • translation.json 파일로 다국어 텍스트를 관리하고 변환 시킴

UX 개선

express를 사용한 api 개발
  • 개선 전: PC 해상도 사이즈로 개발
  • 개선 후: 높은 모바일 점유율과 더불어 손쉽게 등록할 수 있게 하기 위해 모바일 사이즈로 변경
메뉴 위치
  • 개선 전: 상단에 위치
  • 개선 후: 모바일 사이즈에 맞춰 아래쪽에 배치하여 동선 최적화
제품 정보 및 목록
  • 개선 전: 제품명, 등록일, 만료일만 기입
  • 개선 후: 카테고리를 등록하여 관리에 더욱 용이하게 수정
유통기한 임박한 제품
  • 개선 전: 메뉴 게시판명에 알림 표시
  • 개선 후: 메인 상단에 카드형식으로 노출 시켜 진입하자 마자 확인 할 수 있게 하였으며, prograss bar와 D-day 숫자로 얼마 남지 않았음을 직관적으로 알 수 있게 해줌.

얻게된 점

네트워크 확인
  • 페이지에서 새로고침을 하면 계속 데이터가 undefined로 들어오는 이슈가 있었습니다.
  • 임시방편으로 리액트 쿼리 옵션을 refetchOnWindowFocus 로 설정하여 포커싱이 되면 재 렌더링 되게 하였으나, 제대로 된 방법이 아니었습니다.
  • 다음으로 네트워크를 확인했더니 uid가 제대로 들어오지 않았던 문제였습니다. 로그인 정보를 확인할 수 있는 onAuthStateChanged 옵션을 사용하여 app에 조건부 렌더링을 해주었더니 새로고침 했을 때 정상적으로 데이터가 출력되었습니다.
버전 업그레이드에 따른 이슈들
  • 해당 프로젝트는 연초에 진행하였던 것을 react와 firebase의 새로운 버전/문법으로 수정하여 제작한 프로젝트입니다.
  • 그러한 과정에서 여러 에러들을 마주 했는데 문법이 바뀌었을 뿐만 아니라 서로 다른 패키지의 의존성 또한 문제가 되었습니다. 결국 문서를 보고 구글링도 하여 해결하긴 하였는데, 이번 일을 계기로 패키지 의존성과 —force와 --legacy-peer-deps의 차이도 알게 되었습니다.
